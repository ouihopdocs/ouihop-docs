Introduction

An interactor responsible for retrieving surrounding trips that match needed criteria, depending on current state.

This interactor makes use of two SurroundingUsersInteractor's. One that should already exist in the SDK with a short interval time and a quite narrow Geohash filter. Another internal SurroundingUsersInteractor instance is created with a long interval time, but a wider Geohash filter. Both interactors are used to update drivers' locations for surrounding trips.:

startListeningToTrips:

- (void)startListeningToTrips:(NSObject<SurroundingTripsInteractorObserver>*)observer;
Parameters
observer

An observer to be attached, that implements needed protocol. If this observer already exists, nothing happens.

Discussion

* Attaches an observer to the list of interactor observers. If this is the first observer, interactor starts listening for surrounding trips in real-time manner. If an observer is already attached to the interactor, only the new observer is attached.

*

stopListeningToTrips:

- (void)stopListeningToTrips:(NSObject<SurroundingTripsInteractorObserver>*)observer;
Parameters
observer

An observer to be detached. If this observer doesn't exist, nothing happens.

Discussion

* Detaches an observer from the list of interactor observers. If this is the last observer, interactor stops listening for surrounding trips real-time updates. If other observers are still attached to the interactor, the new observer is only attached.

*
